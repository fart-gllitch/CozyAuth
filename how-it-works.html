Authentication Process Breakdown

A detailed look at what happens when a user authenticates with Cozy

1

User Authentication Request

When a user attempts to log in, your application sends credentials to the Cozy DLL, which handles all communication with our secure servers.

// C# Example
using Cozy.Auth;

// Initialize the authentication client
var auth = new CozyClient("YOUR_API_KEY");

// Attempt to authenticate user
var result = await auth.AuthenticateAsync(username, password);
2

Local Encryption

Before any data leaves the user's device, the Cozy DLL encrypts the credentials using AES-256 and a locally generated session key. This ensures that sensitive data is never transmitted in plaintext.

The DLL also adds a cryptographic nonce and timestamp to prevent replay attacks.

3

Edge Server Processing

The encrypted request reaches our nearest edge server, which performs:

  • DDoS protection
  • Rate limiting
  • Traffic analysis for suspicious patterns
  • Request validation

This layer of defense prevents brute force and denial-of-service attacks.

4

Authentication Service

The request is forwarded to our core authentication service, which:

  • Validates the API key
  • Checks subscription status
  • Processes the authentication request

The service uses secure enclaves to decrypt and validate credentials without exposing them to the rest of the system.

5

Zero Knowledge Verification

Our zero-knowledge protocol allows us to verify credentials without ever seeing them in plaintext. We use a cryptographic challenge-response system based on Diffie-Hellman key exchange.

// Zero-knowledge verification pseudocode
                            // Zero-knowledge verification pseudocode
                            Client.generateProof(password) -> passwordProof
                            Server.verifyProof(passwordProof, storedVerifier) -> isValid
                            
                            // No plaintext password is ever transmitted or stored
6

Encrypted Data Storage

User data is stored in our Encrypted Vault using a combination of:

  • AES-256 for symmetric encryption
  • RSA-4096 for asymmetric operations
  • Polymorphic encryption for added security

Each user's data is encrypted with a unique key that is itself encrypted and sharded.

7

Key Sharding

For maximum security, encryption keys are split into multiple shards using Shamir's Secret Sharing algorithm. These shards are distributed across physically separate databases.

This means that a breach of any single database would not compromise user data.

8

Secure Response

After successful authentication, user data is prepared for transmission:

  • Encrypted with session-specific keys
  • Signed for integrity verification
  • Packed with minimal metadata

The response travels back through our edge network to the Cozy DLL.

9

Local Decryption & Session Management

Finally, the Cozy DLL:

  • Decrypts the response using the session key
  • Verifies the digital signature
  • Manages the user session securely
  • Provides access to user data

Key Security Benefits

Why our approach to authentication offers superior protection

Zero Knowledge Architecture

We never see or store plaintext passwords or sensitive user data. Our servers only store cryptographic verifiers that can't be reversed to reveal the original credentials.

Distributed Data Storage

User data and encryption keys are sharded and distributed across multiple physical data centers, ensuring that a breach of any single location can't compromise user accounts.

Military-Grade Encryption

We use AES-256 and RSA-4096, the same encryption standards used by government agencies to protect classified information, ensuring your users' data remains secure.

Ultra-Low Latency

Despite our comprehensive security measures, our authentication takes just 50ms on average, ensuring a smooth user experience without compromising on protection.

Global Edge Network

Our edge servers are strategically located around the world to provide fast authentication regardless of user location, while adding an extra layer of DDoS protection.

Simple Implementation

Our SDKs handle all the complex cryptography behind the scenes, allowing you to implement bank-level security with just a few lines of code in your application.

Frequently Asked Questions

Common questions about our security architecture

What happens if Cozy gets breached?

Even in the unlikely event of a complete breach of our systems, your users' data remains secure. We employ a zero-knowledge architecture where all sensitive data is encrypted with keys that we never have full access to. Additionally, our key-sharding technology splits encryption keys across multiple physically isolated databases, meaning an attacker would need to simultaneously breach multiple facilities to gain access to any data.

How does Cozy compare to OAuth/OIDC?

While OAuth and OpenID Connect are excellent for delegated authorization and single sign-on, Cozy focuses on providing a complete authentication solution with zero-knowledge security. We can work alongside OAuth implementations, adding an extra layer of security to your existing authentication system. Unlike traditional OAuth providers, we never see or store plaintext credentials, and we offer additional features like key sharding and polymorphic encryption.

Can I self-host Cozy?

Yes, for enterprise customers we offer a self-hosted version of Cozy that can be deployed within your own infrastructure. This gives you complete control over your authentication system while still benefiting from our security architecture. Our team can help with the installation and configuration to ensure it meets your specific requirements. Contact our sales team for more information about self-hosting options.

How do password resets work in a zero-knowledge system?

Password resets in a zero-knowledge system are handled through a secure multi-factor verification process. When a user requests a password reset, we send a verification token to their registered email or phone number. Once verified, we allow the creation of a new cryptographic verifier without ever seeing the new password. During this process, we re-encrypt their data with the new key derived from their password. This maintains our zero-knowledge approach while still allowing for account recovery.

What's your compliance and certification status?

Cozy Auth is compliant with major security standards including SOC 2 Type II, GDPR, HIPAA, and PCI DSS. We undergo regular third-party security audits and penetration testing to ensure our systems meet the highest security standards. Our infrastructure is hosted in ISO 27001 certified data centers, and we provide detailed compliance documentation to enterprise customers to assist with their own compliance requirements.

For technical details, cryptographic specifications, and implementation examples, please refer to our technical documentation.
